# Portfolio Website

This repository contains my personal portfolio website. It’s a modern web project built to present my work and my approach to front-end development, both from a technical and design perspective.

Rather than being just a static showcase, the website itself is part of the portfolio. It’s treated as a real application, built with the same care I would apply to a production project: clear structure, modern tools, and attention to maintainability.

## About the project

The site is built with Next.js, which I use to handle routing, rendering, and performance in a clean and scalable way. The entire project is written in TypeScript, mainly to keep the codebase explicit and easier to reason about as it grows.

For styling, I use Tailwind CSS, which allows me to build a consistent UI without relying on large custom stylesheets. Components are based on shadcn/ui, chosen for its simplicity and accessibility, while still leaving full control over styling and composition. Animations and transitions are handled with Framer Motion and are used only where they add clarity or improve the overall experience.

The project is deployed on Netlify, with automatic builds and continuous deployment.

## Structure and design

The design is intentionally minimal and content-focused. The layout is fully responsive and works across different screen sizes without special cases or hacks.

From a code standpoint, the project is organized into small, reusable components, with a clear separation between layout, UI elements, and page sections. This keeps the code readable and makes it easier to iterate on the site over time.

## Why this project exists

This portfolio is meant to show how I think and work as a front-end developer, not just the final visual result.

It reflects my way of approaching problems, choosing tools, and writing code that is easy to understand and evolve. The repository is public because the code itself is part of what I want to show.
